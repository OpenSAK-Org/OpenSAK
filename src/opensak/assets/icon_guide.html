OpenSAK Custom Icon Guide

How to replace OpenSAK's icons with your own — no programming or repackaging required.

Drop replacement SVG files into your custom icons folder (Settings → Advanced → Open icons folder), using the exact file names listed below. Missing or invalid files automatically fall back to OpenSAK's built-in icon — you can never break the app by getting a file wrong.
Restart OpenSAK after adding, changing, or removing an icon file for the change to take effect.

Getting started

Any SVG editor works, but Inkscape (free, open source) is a good place to start and is available directly from your Linux distribution's package manager, e.g.: 

sudo apt install inkscape

Tips for icons that actually render correctly

  • Save as Plain SVG (File → Save As → "Plain SVG"), not Inkscape's default format — the default adds extra inkscape:-namespaced metadata that just bloats the file without adding anything OpenSAK uses.
  • Keep the background transparent — icons are drawn onto a transparent canvas, so a solid background rectangle (unless intentional, like the rounded-square style used by the built-in cache-type icons) will show as a visible box.
  • Stick to basic shapes and paths (<rect>, <circle>, <polygon>, <path>) with simple fill/stroke. Qt's SVG renderer does not support filters (drop shadows, blur) and may not render them at all.
  • Avoid <text> elements that depend on a specific font being installed on the user's system. If you use text in Inkscape, convert it to a path first (Path → Object to Path) so it always renders identically everywhere.
  • Avoid embedding raster images (PNG/JPG) inside the SVG — it works, but bloats the file and looks worse when scaled.
  • Test at the real size. Most of these icons are shown very small in practice (14–32 pixels) — zoom your canvas view to 100% at that pixel size while designing, not just the large editing canvas, so you can judge legibility honestly.
  • Keep your designs original. Colour conventions (e.g. green for Traditional, orange for Multi) are fair game, but don't trace or copy actual Geocaching.com or GSAK artwork or logos.

4. UI icons

icons/ui/<name>.svg — recommended canvas: 16×16 viewBox (can be drawn larger, e.g. 64×64, since SVG scales — just keep it square and simple)

These four have no built-in asset file of their own — the default is a small hand-drawn shape, generated directly in OpenSAK's code. They are all rendered very small (14–16 pixels), as icon-only column headers or cells, so keep designs bold and simple.
